# T-0009: Data Verification Scripts ✅ COMPLETE

**Goal:** Create Python verification scripts to validate CSV export data integrity, coordinate correctness, and session completeness.

**Status:** ✅ Complete

## Deliverables

- `scripts/verify-csv.py` - CSV schema and data validation
- `scripts/verify-alignment.py` - Coordinate and viewport validation  
- `scripts/verify-sessions.py` - Session completeness validation
- `scripts/requirements-analysis.txt` - Python dependencies
- `scripts/README.md` - Usage documentation

## Acceptance Criteria ✅

### verify-csv.py
- [x] All 20 required columns present
- [x] No missing critical values (session_id, slide_id, event, zoom_level, dzi_level)
- [x] Timestamps in chronological order per session
- [x] DZI level matches zoom level (2.5×→14, 5×→15, etc.)
- [x] Viewport bounds within slide dimensions
- [x] Exit code 0 if valid, 1 with error report if invalid

### verify-alignment.py
- [x] Click coordinates (`click_x0`, `click_y0`) fall within slide bounds
- [x] Viewport bounds contain the center point
- [x] Center coordinates consistent with DZI level
- [x] Generate report with validation results

### verify-sessions.py
- [x] Each session has `app_start` event (first event)
- [x] Each session has `slide_load` event
- [x] Completed sessions have `slide_next` event with label
- [x] Event sequences are logical
- [x] Calculate per-session statistics: duration, click count, zoom distribution

## Usage

```bash
python scripts/verify-csv.py pathology_events.csv
python scripts/verify-alignment.py pathology_events.csv --manifest tiles/test_slide_files/manifest.json
python scripts/verify-sessions.py pathology_events.csv
```

## Notes

Cell indices (i, j) have been removed from event logging. Scripts now validate exact click coordinates (`click_x0`, `click_y0`) which provide all needed information for patch extraction.
